# Formulas de Copy

Frameworks para transformar contexto em copy que converte.

---

## PAS - Problem Agitate Solution

**Uso:** Headlines, secoes de abertura

### Estrutura

```
1. PROBLEM - Identifique a dor
2. AGITATE - Amplifique a consequencia
3. SOLUTION - Apresente a saida
```

### Template

```markdown
[DOR] esta te custando [CONSEQUENCIA].
[PRODUTO] resolve isso em [TEMPO/FORMA].
```

### Exemplos

| Contexto | Copy PAS |
|----------|----------|
| CRM para vendas | "Leads esfriando no Excel? Cada dia sem follow-up e dinheiro perdido. EasyCRM organiza seu funil em 5 minutos." |
| Gestao de projetos | "Projetos atrasados de novo? Retrabalho custa 3x mais que prevenir. TaskFlow mostra gargalos antes que explodam." |
| Automacao | "Copiando dados entre sistemas? Horas perdidas todo dia. Zapify conecta tudo em 2 cliques." |

## BAB - Before After Bridge

**Uso:** Storytelling, secoes de transformacao

### Estrutura

```
1. BEFORE - Situacao atual (com dor)
2. AFTER - Situacao desejada (com beneficio)
3. BRIDGE - Como chegar la (seu produto)
```

### Template

```markdown
Antes: [SITUACAO ATUAL COM DOR]
Depois: [SITUACAO DESEJADA]
Ponte: [PRODUTO] faz essa transicao.
```

### Exemplos

| Contexto | Copy BAB |
|----------|----------|
| Dashboard | "Antes: relatorios manuais toda segunda. Depois: dados ao vivo no celular. DataViz e a ponte." |
| Onboarding | "Antes: clientes perdidos no primeiro uso. Depois: ativacao em 24h. OnboardKit e a ponte." |
| Suporte | "Antes: tickets se acumulando. Depois: 80% resolvidos por bot. HelpDesk AI e a ponte." |

## 4Us - Validacao de Headlines

**Uso:** Testar se headline e forte o suficiente

### Criterios

| U | Pergunta | Se NAO |
|---|----------|--------|
| **Urgent** | Cria senso de urgencia? | Adicione consequencia de nao agir |
| **Unique** | E diferente do que outros dizem? | Especifique seu diferencial |
| **Ultra-specific** | Tem numeros/detalhes concretos? | Adicione stats, tempo, % |
| **Useful** | Mostra beneficio claro? | Foque no resultado, nao na feature |

### Aplicacao

```markdown
## Headline Original
"Gerencie seus projetos facilmente"

## Teste 4Us
- Urgent? NAO - nao ha consequencia
- Unique? NAO - qualquer ferramenta diz isso
- Ultra-specific? NAO - "facilmente" e vago
- Useful? FRACO - "gerenciar" nao e beneficio

## Headline Melhorada
"Pare de perder deadlines. Setup em 5 min, sem treinamento."

## Teste 4Us
- Urgent? SIM - "perder deadlines" e consequencia
- Unique? SIM - "sem treinamento" e diferencial
- Ultra-specific? SIM - "5 min"
- Useful? SIM - beneficio claro (nao perder deadlines)
```

## F-B-O - Feature Benefit Outcome

**Uso:** Transformar features tecnicas em argumentos de venda

### Estrutura

```
FEATURE: O que o produto FAZ (tecnico)
BENEFIT: O que isso SIGNIFICA para o usuario
OUTCOME: O RESULTADO na vida/trabalho do usuario
```

### Template

```markdown
[FEATURE] para que voce [BENEFIT], resultando em [OUTCOME].
```

### Tabela de Conversao

| Feature | Benefit | Outcome |
|---------|---------|---------|
| Dashboard em tempo real | Ver metricas ao vivo | Tomar decisoes mais rapidas |
| Automacoes no-code | Criar fluxos sem programar | Economizar horas por semana |
| Integracao com Slack | Receber alertas onde trabalha | Nunca perder notificacao importante |
| Relatorios automaticos | Nao fazer relatorio manual | Ter segundas-feiras livres |
| Multi-workspace | Separar projetos por cliente | Manter organizacao mesmo crescendo |
| Historico de alteracoes | Saber quem mudou o que | Resolver conflitos em segundos |

### Exemplo Completo

```markdown
## Feature: API RESTful com webhooks

**Errado (so feature):**
"API RESTful completa com suporte a webhooks"

**Melhor (feature + benefit):**
"Integre com qualquer ferramenta que voce ja usa"

**Ideal (feature + benefit + outcome):**
"Conecte seu stack inteiro em uma tarde. Seus dados fluem automaticamente entre sistemas — sem copiar e colar nunca mais."
```

---

## Quando Usar Cada Formula

| Situacao | Formula | Por que |
|----------|---------|---------|
| Headline principal | PAS | Captura atencao pela dor |
| Secao "Como funciona" | BAB | Mostra transformacao |
| Validar headline | 4Us | Garante que e forte |
| Listar features | F→B→O | Transforma tecnico em beneficio |
| Testimonials | BAB | Cliente conta transformacao |
| CTA section | PAS | Urgencia para agir |

## Combinando Formulas

**Headline (PAS):**
> "Leads esfriando no Excel? EasyCRM organiza seu funil em 5 min."

**Subtitle (F→B→O):**
> "Pipeline visual + automacoes para voce fechar mais, digitando menos."

**CTA section (BAB):**
> "Saia do caos das planilhas. Tenha controle total do seu funil. Comece gratis hoje."
